# 开发者界面重构设计文档

> 创建日期：2026-04-02

## 一、背景与目标

### 当前问题
- 列表页信息密度低，概览区统计指标无意义
- 详情页展示大量管理者不关心的个人信息
- 三级路由冗余，应简化为两级
- 卡片颜色花哨但无明确意义

### 设计目标
- **管理者视角**：快速了解团队成员工作状态，清晰看到并行任务分布
- **信息密度优化**：每个展示元素都有明确意义
- **两级路由**：列表页（概览）→ 详情页（时间轴流水线）

---

## 二、页面结构

### 2.1 列表页 `/developers`

#### 概览区统计指标

| 指标 | 说明 | 意义 |
|------|------|------|
| 总开发者 | 团队总人数 | 团队规模 |
| 进行中需求 | 正在开发的需求总数 | 当前工作负载 |
| 等待确认 | 卡在方案评审等确认环节的需求 | 潜在瓶颈 |
| 已完成 | 本周完成的需求数 | 产出效率 |
| 平均饱和度 | 团队整体工作饱和程度 | 资源利用 |

#### 开发者卡片信息

保留：
- 姓名、职位标签
- 进行中项目数
- 工作状态（饱和/正常/空闲）
- 最近活动时间

移除：
- 步骤进度（与整体进度重复）
- 工号（非第一眼需要）

新增：
- "添加成员"按钮（列表页顶部）

### 2.2 详情页 `/developers/[id]`

#### 页面结构

```
┌─────────────────────────────────────────────────────────────┐
│ ← 返回    张明  [全栈工程师]    ●在线    工作状态：饱和      │
├─────────────────────────────────────────────────────────────┤
│ 统计概览                                                     │
│ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│ │ 进行中 3  │ │ 等待确认 1│ │ 本周完成 2│ │ 平均进度  │        │
│ │ 个需求    │ │ 个需求    │ │ 个需求    │ │ 72%      │        │
│ └──────────┘ └──────────┘ └──────────┘ └──────────┘        │
├─────────────────────────────────────────────────────────────┤
│ 时间维度切换：[日] [周] [历史]                               │
├─────────────────────────────────────────────────────────────┤
│ 时间轴流水线区域                                             │
│ （默认展示当天，可切换周/历史）                               │
└─────────────────────────────────────────────────────────────┘
```

#### 时间轴流水线视图

**日视图**：展示当天每个需求的阶段进度
**周视图**：展示一周内每天的需求推进情况
**历史视图**：已完成需求列表 + 统计摘要

## 三、颜色规范

### 3.1 需求状态颜色

| 状态 | 颜色 | CSS变量/类 |
|------|------|------------|
| 进行中 | 紫色 | `primary` |
| 等待确认 | 琥珀色 | `bg-amber-500` |
| 已完成 | 绿色 | `bg-emerald-500` |
| 阻塞 | 红色 | `bg-red-500` |

### 3.2 职位标签颜色

| 职位 | 颜色 | 渐变类 |
|------|------|--------|
| 全栈工程师 | 蓝色 | `from-blue-500 to-cyan-500` |
| 后端工程师 | 绿色 | `from-green-500 to-emerald-500` |
| 前端工程师 | 橙色 | `from-orange-500 to-amber-500` |
| UI设计师 | 粉色 | `from-pink-500 to-rose-500` |
| 产品经理 | 紫色 | `from-violet-500 to-purple-500` |

### 3.3 工作状态颜色

| 状态 | 颜色 | 说明 |
|------|------|------|
| 饱和 | 红色 | 负载过重 |
| 正常 | 绿色 | 正常工作 |
| 空闲 | 灰色 | 可接新任务 |

## 四、交互设计

### 4.1 列表页

1. 点击开发者卡片 → 进入详情页
2. 点击"添加成员"按钮 → 弹出添加成员对话框
3. 搜索功能：按姓名、职位搜索

### 4.2 详情页

1. 时间维度切换：日/周/历史三个Tab
2. 日视图：展示当天任务卡片，可点击跳转到需求详情
3. 周视图：横向滚动的时间轴，展示一周工作分布
4. 历史视图：列表展示已完成需求，可筛选时间范围

### 4.3 流水线任务卡片

- 显示需求名称
- 显示当前阶段
- 显示协作AI专家
- 点击跳转到需求详情页

## 七、技术方案

### 7.1 技术栈
- Next.js 16 App Router
- React 19
- TypeScript 5
- Tailwind CSS 4
- shadcn/ui

### 7.2 路由结构
```
/developers          → 列表页
/developers/[id]     → 详情页（删除原有的三级路由）
```

### 7.3 组件拆分
- `DeveloperCard` - 开发者卡片
- `AddMemberDialog` - 添加成员对话框
- `TimelineView` - 时间轴流水线（支持日/周/历史三种模式）
- `TaskCard` - 流水线任务卡片
